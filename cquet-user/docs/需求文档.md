# 需求文档

> 本文档是项目的需求基线，描述「做什么」和「为什么」。架构细节请见 `docs/架构文档.md`。

---

## 1. 项目背景

**项目名称**：重庆文旅用户端
**项目概述**：为重庆文旅网站构建独立的用户端界面，面向游客和本地居民，提供景区、场馆、活动的一站式浏览、报名和互动体验。

---

## 2. 目标用户

| 用户角色 | 需求 |
|---------|------|
| 游客 | 发现重庆景点、场馆、活动，阅读文旅资讯，报名参与，发表评论 |
| 本地居民 | 查找身边的文化场馆、活动，收藏感兴趣的内容 |
| 管理者 | 发布文旅资讯和公告，了解用户参与情况 |

---

## 3. 核心价值

**高端质感 + 山城内容** — 用做 luxury hotel 网站的手法做旅游平台。

系统应该让用户感受到：这是一个认真做过的、品质有追求的旅游平台，而不仅仅是一个课程作业。

---

## 4. 功能范围

### 4.1 页面清单

| 页面 | 功能 | 路由 |
|------|------|------|
| 首页 | 全屏 Hero + 精选推荐 + 关于重庆 + 五栏入口 + 最新资讯/公告 | `/` |
| 景区列表 | 卡片列表 + 三级筛选（名称/城市/等级） | `/spots` |
| 景区详情 | 多图轮播 + 信息 + 收藏 + 关联场馆 | `/spots/:id` |
| 场馆列表 | 卡片列表 + 二级筛选（名称/地址） | `/venues` |
| 场馆详情 | 多图轮播 + 设施 + 近期活动 | `/venues/:id` |
| 活动列表 | 卡片列表 + 三级筛选（名称/场馆/状态） | `/activities` |
| 活动详情 | 活动介绍 + 报名入口 + 评论区 | `/activities/:id` |
| 新闻列表 | 资讯列表 + 日期筛选 | `/news` |
| 新闻详情 | 资讯全文 | `/news/:id` |
| 公告列表 | 公告列表（公开） | `/notices` |
| 公告详情 | 公告全文 | `/notices/:id` |
| 登录 | JWT token 认证 | `/login` |
| 注册 | 用户创建 + 自动登录 | `/register` |
| 用户中心 | 资料 + 收藏 + 报名 + 评论 | `/profile` |

### 4.2 业务功能详解

#### 景区详情 — 收藏
- 点击「收藏」→ POST 收藏接口 → 按钮变为「已收藏」
- 再次点击→ 取消收藏
- 未登录时点击 → 跳转登录页

#### 活动详情 — 报名
1. 点击「立即报名」→ 确认弹窗（活动名 + 时间 + 地点）
2. 确认 → POST `/tourism/activity-application`
3. 跳转「我的报名」，状态显示「待审批」

#### 活动详情 — 评论区
- 活动详情页底部展示评论列表
- 发表评论：内容 + 提交 → POST `/tourism/comment`
- 支持回复（parentId）
- 评论显示：用户名 + 时间 + 内容

#### 用户中心 — 我的收藏
- Tab 切换：全部 / 景区 / 场馆 / 活动
- 卡片列表（图片 + 名称 + 类型）
- 取消收藏按钮

#### 用户中心 — 我的报名
- 报名记录列表：活动名 + 时间 + 审批状态
- 状态标签：待审批（黄）/ 已通过（绿）/ 已拒绝（红）+ 原因

#### 用户中心 — 我的评论
- 我发表的评论列表
- 可查看回复

## 5. 交互质量标准

### 5.1 页面状态处理

每个列表页和详情页必须处理以下状态：

| 状态 | 处理方式 |
|------|----------|
| 加载中 | 骨架屏（Skeleton），不显示白屏 |
| 加载失败 | 错误提示 + 重试按钮 |
| 空数据 | 友好的空状态插画 + 引导文案 |
| 网络断开 | Toast 提示「网络连接已断开」 |

### 5.2 动画与过渡

| 场景 | 行为 |
|------|------|
| 页面进入 | 内容渐入，stagger 50ms |
| 按钮点击 | scale: 0.97，按下反馈 |
| 卡片 hover | scale: 1.02，阴影加深 |
| 页面切换 | 轻微 fade 过渡 |
| 图片加载 | 淡入效果（loading="lazy" + opacity 动画）|

### 5.3 表单交互

- 输入框 focus 时边框高亮
- 表单提交显示 loading 状态（按钮禁用 + spinner）
- 提交后根据结果显示成功/失败 Toast
- 错误信息紧跟对应字段，不统一堆在顶部

**参考网站**：[Range Developments](https://www.rangedevelopmentsgroup.com/)（高端地产官网）

| 要素 | 值 |

| 原子 | 规格 |

| 栅格 | 12列，gutter 24px |
| 内边距 | 移动端 16px，桌面端 40px |

## 7. 移动端响应式

| 断点 | 宽度 |
|------|------|
| Desktop | ≥ 1024px |
| Tablet | 768px–1023px |
| Mobile | < 768px |

**Mobile 行为**：
- Nav：Logo + 汉堡菜单（全屏 overlay）
- Hero：保持全屏，文字缩小
- 精选推荐：单列 + 左右滑动
- 卡片网格：3列 → 2列 → 1列
- Footer：4列 → 2列
