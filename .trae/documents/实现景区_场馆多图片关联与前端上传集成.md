## 目标
- 景区与场馆均可关联多张图片（多对多）
- 新增/修改时支持批量添加图片 URL，自动建图记录并建立关联
- 编辑时拉取已有关联图片，前端“图片上传”组件展示与可增删

## 后端接口（在现有风格基础上新增）
### 图片记录管理（已存在）
- `POST /tourism/images`：新增单条图片（URL） 权限：`tourism:image:add`
- `POST /tourism/images/batch`：批量新增图片（URL列表） 权限：`tourism:image:add`
- `GET /tourism/images/{id}`：查询图片 权限：`tourism:image:query`
- `DELETE /tourism/images/{ids}`：删除图片 权限：`tourism:image:remove`

### 景区图片关联
- `GET /tourism/scenic-spot/{id}/images`：返回关联图片列表 权限：`tourism:scenic-spot:image:list`
- `PUT /tourism/scenic-spot/{id}/images`：覆盖式设置关联图片（请求体为 `imageIds: number[]`） 权限：`tourism:scenic-spot:image:edit`
- 服务层新增：
  - `List<TourismImage> getImagesByScenicSpot(Long scenicSpotId)`
  - `boolean setImagesForScenicSpot(Long scenicSpotId, List<Integer> imageIds)`（先删后插，事务，`sort` 默认 0）
- 关联表：`tourism_scenic_spot_image`（已存在）

### 场馆图片关联
- `GET /tourism/venue/{id}/images`：返回关联图片列表 权限：`tourism:venue:image:list`
- `PUT /tourism/venue/{id}/images`：覆盖式设置关联图片（`imageIds: number[]`） 权限：`tourism:venue:image:edit`
- 服务层新增：
  - `List<TourismImage> getImagesByVenue(Long venueId)`
  - `boolean setImagesForVenue(Long venueId, List<Integer> imageIds)`（先删后插，事务，`sort` 默认 0）
- 关联表：`tourism_venue_image`（已存在）

### 权限建议（若依菜单/按钮）
- `tourism:scenic-spot:image:list`、`tourism:scenic-spot:image:edit`
- `tourism:venue:image:list`、`tourism:venue:image:edit`
- 可选简化：若你希望减少权限项，可将上述两个“list/编辑”合并复用主表的 `query/edit` 权限

## 前端改造
### 通用 API
- 新增：
  - `listScenicSpotImages(id)`、`setScenicSpotImages(id, imageIds)`（在 `src/api/tourism/scenicSpot.js`）
  - `listVenueImages(id)`、`setVenueImages(id, imageIds)`（在 `src/api/tourism/scenicVenue.js`）
  - 已有批量图片创建：`batchUploadImageUrls(urls)`（`src/api/tourism/uploadPictures.js`）

### 景区页面 `src/views/tourism/scenicSpot/index.vue`
- 重新启用已注释的 `<image-upload>` 组件：
  - 表单字段：`imageUrls: string[]`
  - 新增/修改提交流程：
    1. 从 `<image-upload>` 获取 URL 列表
    2. 调用 `batchUploadImageUrls(urls)` → 返回图片记录（含 `id`）
    3. 调用 `setScenicSpotImages(scenicSpotId, imageIds)` 建立关联
  - 编辑打开时：
    - 调用 `listScenicSpotImages(id)`，将返回的图片列表中的 `url` 显示在 `<image-upload>` 中

### 场馆页面 `src/views/tourism/venue/index.vue`
- 加入 `<image-upload>` 并遵循同景区的流程：
  - 表单字段：`imageUrls: string[]`
  - 新增/修改提交：`batchUploadImageUrls` → `setVenueImages(venueId, imageIds)`
  - 编辑打开：`listVenueImages(id)` 预填

### 交互细节
- 删除图片关联：
  - 前端在 `<image-upload>` 中移除某张图片 → 在提交时不含该 URL → `set*Images` 覆盖式生效（旧的不再写入）
  - 独立删除图片记录（`DELETE /tourism/images/{ids}`）仅用于清理孤立图片（可选）
- 排序：
  - `<image-upload>` 返回的顺序按数组索引 `sort` 写入关联表；前端按同序展示

## 验证
- 新增景区/场馆时，选择多张图片 URL，保存后列表或详情可查看已关联图片
- 编辑景区/场馆时，已关联图片正确回填，支持增删并覆盖更新
- 权限控制：只有具备 `*image:edit` 的用户可修改关联；具备 `*image:list` 的用户可查看
