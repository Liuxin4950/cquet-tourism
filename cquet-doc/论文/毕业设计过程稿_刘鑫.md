# 摘  要

本文围绕重庆文旅信息展示与后台管理需求，设计并实现了一个基于 Spring Boot 的重庆文旅网站。系统基于若依框架进行二次开发，整体采用前后端分离架构，后端使用 Spring Boot、Spring Security、MyBatis-Plus 和 MySQL 实现，前端分为管理端和用户端两部分。系统主要实现了景区管理、场馆管理、特色活动管理、新闻公告管理、图片管理和数据大屏展示等功能。针对开发过程中管理端接口与用户前台接口职责不清的问题，系统进一步对旅游业务接口进行了 `admin` 与 `web` 分层设计，提高了系统结构清晰度和可维护性。测试结果表明，系统主要功能运行稳定，能够满足重庆文旅信息发布、后台内容维护和用户前台浏览展示的基本需求。

**关键词：** Spring Boot；若依框架；文旅网站；前后端分离；Vue；MySQL

# ABSTRACT

This paper designs and implements a Chongqing cultural tourism website based on Spring Boot for tourism information display and background management. The system is developed based on the RuoYi framework and adopts a front-end and back-end separation architecture. The back-end is implemented with Spring Boot, Spring Security, MyBatis-Plus and MySQL, while the front-end consists of an admin side and a user side. The system mainly realizes scenic spot management, venue management, featured activity management, news and notice management, image management, and data dashboard display. In order to solve the problem of unclear responsibilities between admin APIs and user-side APIs during development, the tourism interfaces are further divided into `admin` and `web` layers, which improves the clarity and maintainability of the system structure. Test results show that the main functions of the system run stably and can meet the basic needs of Chongqing cultural tourism information release, background content maintenance, and user-side browsing.

**Keywords:** Spring Boot; RuoYi framework; cultural tourism website; front-end and back-end separation; Vue; MySQL

# 1 绪论

## 1.1 选题背景与意义

随着数字经济和智慧旅游建设的不断推进，传统的文旅信息传播和管理方式已经难以满足用户对便捷化、集中化和智能化服务的需求。重庆作为西南地区的重要旅游城市，拥有丰富的自然景区、历史文化资源、特色活动和公共文旅场馆。若这些资源缺乏统一的信息发布平台，不仅用户获取信息效率较低，也会增加管理人员在内容维护、数据更新和活动审核方面的工作成本。因此，设计并实现一个面向重庆文旅场景的网站系统，既具备现实意义，也具有较强的工程实践价值。

### 1.1.1 选题背景

近年来，基于 Web 的信息管理系统已经成为各类业务数字化建设的重要形式。文旅类网站不再仅仅满足于展示静态页面，而是逐步朝着内容统一管理、前后端分离、数据可视化和智能服务拓展等方向发展。对于重庆文旅信息服务而言，景区信息、场馆信息、活动资讯和新闻公告等内容如果仍然分散在不同平台，不仅不利于用户集中查询，也会影响管理部门对内容的统一维护和运营决策。

本课题以重庆文旅网站为对象，结合当前主流的 Spring Boot 与 Vue 技术栈，设计并实现一个集后台管理与前台展示于一体的系统。项目以后端若依框架为基础进行二次开发，在继承其用户权限管理、系统管理和基础框架能力的同时，扩展实现重庆文旅相关业务功能。通过该系统，管理员可以对景区、场馆、活动、新闻公告和图片等数据进行统一维护，普通用户则可以在前台对相关信息进行浏览查询，从而提升系统的信息服务能力和管理效率。

### 1.1.2 选题意义

本课题具有以下几个方面的意义。

第一，具有实际应用意义。系统能够为游客和本地居民提供统一的重庆文旅信息查询平台，使景区、场馆、活动、新闻公告等内容能够集中展示，提高信息获取效率和浏览体验。

第二，具有管理优化意义。通过后台管理端，管理员能够对文旅相关数据进行统一维护，实现景区、场馆、活动和资讯内容的规范化管理，并借助数据大屏查看系统统计信息，提高后台管理效率。

第三，具有工程实践意义。本课题基于若依框架二次开发，结合 Spring Boot、Vue、MyBatis-Plus 和 MySQL 等主流技术实现完整项目，有助于提高本人在需求分析、系统设计、接口开发、前后端联调、测试验证和问题修复等方面的综合能力。

第四，具有扩展意义。系统当前已经形成较完整的文旅业务基础，后续可以在现有架构上继续接入基于 Python 智能体的路线规划和个性化推荐功能，使项目具备进一步扩展的可能性。

## 1.2 项目的主要内容

本项目主要实现了一个基于 Spring Boot 的重庆文旅网站，整体采用前后端分离架构，系统分为管理端和用户端两个部分。管理端主要用于文旅数据的维护和管理，用户端主要面向普通游客和网站用户提供信息展示与浏览服务。

在系统功能方面，管理端实现了景区管理、场馆管理、特色活动管理、新闻公告管理、图片管理和数据大屏统计展示等功能。管理员可以通过后台完成相关数据的新增、修改、删除和查询操作，并对系统中活动、资讯和展示内容进行统一维护。用户端实现了景区列表与详情展示、场馆列表与详情展示、活动列表与详情展示、新闻公告浏览、登录注册和个人中心等功能。

在系统结构方面，项目后端采用 Maven 多模块方式组织，包括 `cquet-admin`、`cquet-framework`、`cquet-system`、`cquet-tourism`、`cquet-common` 等模块。前端分为 `cquet-ui-demo` 管理端项目和 `cquet-user` 用户端项目。为了进一步规范系统接口设计，在开发过程中还对旅游业务接口进行了 `admin` 与 `web` 分层处理，使管理端接口和用户前台接口按照使用对象进行区分。

在本次毕业设计中，本人主要负责系统后端业务模块开发、接口设计与调整、部分数据库结构整理、部分用户前端联调以及系统测试等工作。围绕“基于 Spring Boot 的重庆文旅网站的设计与实现”这一课题，本人结合项目整体功能需求，重点完成了景区管理、场馆管理、特色活动管理、新闻公告管理和数据大屏展示等模块的后端实现工作，并参与了用户前端接口重新对接和系统结构优化工作。

在项目开发过程中，本人承担的主要任务包括以下几个方面：第一，参与系统需求分析，梳理景区、场馆、活动、新闻公告和数据展示等模块的功能需求；第二，完成旅游业务模块中 Controller、Service、Mapper 等核心代码编写；第三，参与数据库表结构整理，使核心业务表能够支持系统当前功能实现；第四，参与用户前端与后端接口联调，对前台调用接口进行统一调整；第五，参与系统测试、构建验证和问题修复；第六，参与项目文档与毕业论文草稿的整理工作。

此外，在用户前端接口重对接阶段，本人完成了景区、场馆、活动、新闻和公告等模块接口路径的统一调整，使用户前台能够通过 `/web/tourism/**` 路径获取数据。通过这一系列工作，不仅保证了系统当前阶段的正常运行，也为项目后续扩展、答辩展示和论文撰写提供了更加稳定和规范的基础。

# 2 开发技术简介

本项目采用前后端分离架构，后端负责业务逻辑处理、权限认证和数据持久化，前端负责页面展示、用户交互和接口调用。系统整体采用若依框架和 Spring Boot 作为后端开发基础，前端分别使用 Vue 2 和 Vue 3 技术实现管理端和用户端页面，以 MySQL 作为主要数据存储手段，从而支撑重庆文旅网站的业务实现。

## 2.1 后端开发技术简介

本项目后端主要采用 Spring Boot 作为核心开发框架。Spring Boot 具有自动配置、开发效率高、生态完善等优点，能够快速完成项目搭建和接口开发。系统在 Spring Boot 基础上结合若依框架进行二次开发，充分利用了若依在权限管理、系统管理、代码组织和后台开发方面的优势。

在 Web 层方面，项目使用 Spring MVC 处理前端请求，通过 Controller 对外提供 RESTful 风格接口。权限控制方面，系统使用 Spring Security 对管理端访问进行认证和授权，并结合 JWT 完成令牌校验，保证接口访问安全性。数据访问层方面，系统主要采用 MyBatis-Plus 完成数据库的增删改查操作。MyBatis-Plus 在保留 MyBatis 灵活性的同时，减少了重复代码，提高了开发效率。整个项目以后端多模块方式组织，使系统结构更加清晰，便于后续维护和扩展。

## 2.2 前端开发技术简介

本项目前端分为管理端和用户端两部分。管理端采用 Vue 2 和 Element UI 开发，用于实现文旅后台管理页面。Vue 2 在后台管理系统开发中应用较为广泛，配合 Element UI 能够快速构建结构清晰、交互完整的业务管理界面。

用户端采用 Vue 3、Vite 和 TailwindCSS 开发，主要实现景区、场馆、活动、新闻和公告等内容的浏览展示。Vue 3 具备更加灵活的组合式开发方式，Vite 具有更快的开发和打包速度，TailwindCSS 则可以提高页面样式开发效率。除此之外，项目前端还使用 Axios 进行接口通信，使用 Vue Router 完成页面路由管理，并通过 Pinia 管理部分前端状态数据。

## 2.3 数据库简介

本项目数据库采用 MySQL。MySQL 是一种广泛应用的关系型数据库管理系统，具有稳定性高、性能较好、易于部署和维护等优点，能够满足本项目文旅信息管理和展示系统的数据存储需求。

根据系统功能需求，项目建立了景区表、场馆表、活动表、活动审批表、新闻表、公告表、图片表和用户表等核心业务数据表。其中，景区表用于保存景区基础信息，场馆表用于保存各类文旅场馆信息，活动表和活动审批表用于记录特色活动数据及审核流程，新闻表和公告表用于信息发布，图片表用于统一维护系统中的图片资源。这些数据表共同支撑了系统核心业务功能的实现。

## 2.4 中间件介绍

本项目在开发中主要使用了 Druid 数据库连接池和 JWT 认证机制。Druid 用于管理数据库连接，提高数据库访问效率并支持运行监控。JWT 用于实现系统的无状态认证，在用户登录后生成令牌，并在后续请求中完成身份校验，适合前后端分离系统的认证场景。

目前项目重点在于核心业务功能开发和系统结构规范化，后续如果进一步优化系统性能，还可以结合 Redis 等中间件实现热点数据缓存、登录状态优化和高频数据访问加速。

# 3 系统分析与设计

本系统围绕重庆文旅网站的实际业务需求展开分析与设计，主要面向两类用户：一类是系统管理员，负责后台内容维护和管理；另一类是普通用户，负责前台浏览和信息查询。根据需求分析结果，系统划分为多个核心功能模块，并结合业务对象之间的关系设计对应数据库表结构，以支撑系统实现。

## 3.1 系统功能分析与流程设计

### 3.1.1 景区管理模块

（1）关键需求描述  
景区管理模块主要用于维护重庆 A 级景区相关信息。管理员可以在后台新增、修改、删除和查询景区信息，包括景区名称、等级、所属地区、详细介绍、图片等内容。普通用户则可以在用户前台查看景区列表、景区详情和景区图片信息。

（2）业务流程图  
管理员登录后台后，进入景区管理页面，填写景区信息并提交保存，系统将数据写入数据库。用户前台通过景区列表接口和详情接口获取数据，并在页面中完成展示。

（3）ER图  
该模块涉及景区实体和图片实体，景区与图片之间存在关联关系，一个景区可以关联多张图片。

### 3.1.2 场馆管理模块

（1）关键需求描述  
场馆管理模块主要用于维护重庆文旅场馆信息，包括场馆名称、场馆类别、地址、开放信息和关联图片等。管理员可通过后台维护场馆数据，用户可在前台查看场馆列表、场馆详情和场馆相关活动信息。

（2）业务流程图  
管理员在后台录入或修改场馆信息，系统将场馆数据保存到数据库中。用户进入场馆页面后，通过接口获取场馆列表和详情数据，并进一步查看场馆关联的活动和图片。

（3）ER图  
该模块涉及场馆实体、图片实体和活动实体，场馆与图片之间存在关联关系，场馆与活动之间存在一对多关系。

### 3.1.3 新闻公告管理模块

（1）关键需求描述  
新闻公告管理模块用于统一维护系统中的新闻动态和通知公告。管理员可以在后台新增、修改、删除新闻和公告内容，用户前台可以查看新闻列表、公告列表和详情内容。

（2）业务流程图  
管理员在后台发布新闻或公告信息，系统将内容保存到对应数据表。用户端页面调用新闻公告接口获取数据，并按列表和详情形式进行展示。

（3）ER图  
该模块主要涉及新闻实体和公告实体，两者均属于独立信息发布对象。

### 3.1.4 特色活动管理模块

（1）关键需求描述  
特色活动管理模块用于维护重庆文旅特色活动信息。管理员可以新增、修改、删除活动数据，控制活动状态，并对活动信息进行审核管理。用户端可以查看活动列表、活动详情和活动状态等内容。

（2）业务流程图  
管理员在后台创建活动并保存活动信息，系统将活动数据存入数据库。若活动需要审核，则由管理员在审批模块中进行处理。用户端通过活动接口读取已经发布的活动信息并进行展示。

（3）ER图  
该模块主要涉及活动实体、场馆实体和审批实体。一个场馆可以包含多个活动，一个活动可以对应多条审批记录。

### 3.1.5 数据大屏模块

（1）关键需求描述  
数据大屏模块主要面向管理端展示目前系统的整体概况，管理人员可以看到重庆各地区相关数据分布、景区热度排行、数量变化趋势以及系统总体聚合信息。

（2）业务流程图  
管理员登录后台后进入数据大屏页面，系统通过聚合查询接口返回统计结果，前端页面将系统总体数据、区域分布和热度趋势通过图表方式展示出来。

（3）ER图  
该模块主要涉及多个业务表的聚合查询以及地图展示数据的整合处理，依赖景区、活动、场馆等多个对象的统计结果。

## 3.2 数据库表设计

根据系统功能需求，本项目设计了多个核心业务数据表，包括景区表、场馆表、活动表、活动审批表、新闻表、公告表、图片表和用户表等。数据库设计过程中，主要围绕文旅信息展示和后台内容管理两大业务方向展开，尽量保证数据结构清晰、关系明确、便于维护。设计过程中遵循了数据库的三范式，保证了数据的原子性和结构规范性。

## 5.1 测试用例设计方法

本系统测试过程中，主要采用以下几种测试方法。

1. 等价划分法：将输入数据划分为有效等价类和无效等价类，并从中选取代表性数据进行测试，用于验证系统对不同类型输入的处理能力。  
2. 边界值划分法：针对标题长度、必填项限制、编号范围和状态值等边界条件设计测试用例，用于检查系统在边界情况下是否能够正常处理数据。  
3. 错误推测法：根据开发与使用经验，对系统中可能出现问题的地方进行重点测试，例如重复名称新增、空值提交、错误路径访问、缺少必填参数等情况。  
4. 场景法：模拟管理员维护数据、活动审核、用户浏览景区和查看新闻公告等真实使用场景，验证系统整体业务流程是否能够顺利完成。  

在具体执行过程中，先对后台管理模块的新增、修改、删除、查询等核心功能进行验证，再对用户前台展示接口进行联调测试，最后对数据大屏的统计结果和图表展示效果进行检查。对于每个模块，测试时均从正常流程、异常输入和结果展示三个方面进行分析，尽量保证测试内容覆盖系统主要使用场景。

# 6 结论

本文完成了基于 Spring Boot 的重庆文旅网站的设计与实现。系统围绕重庆文旅业务需求，构建了包含管理后台和用户前台的完整网站系统，实现了景区管理、场馆管理、特色活动管理、新闻公告管理和数据大屏展示等主要功能，基本达到了课题预期目标。

在开发过程中，本人不仅完成了核心业务模块的后端开发和部分前端联调工作，还对系统接口结构进行了规范化改造，将管理端接口和用户前台接口按 `admin` 与 `web` 两类进行分层整理，从而提高了系统的可维护性和扩展性。通过测试，系统主要功能运行正常，能够较好地满足重庆文旅信息展示和后台内容管理的基本需求。

当然，本系统仍存在一些不足，例如用户前台交互体验仍可继续优化，后续智能推荐和路线规划功能尚未正式接入，系统测试的广度与深度也可以继续加强。今后可以在现有系统基础上进一步完善前台体验、扩展数据分析能力，并结合 Python 智能体实现个性化推荐和路线规划功能。总体而言，本次毕业设计提高了本人在系统分析、架构设计、接口开发、前后端联调和测试验证等方面的综合能力，具有较好的实践意义。
