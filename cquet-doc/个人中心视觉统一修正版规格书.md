# 个人中心视觉统一修正版规格书

项目：`cquet-user`
 页面：`/profile`
 目标：在不破坏现有主站设计语言的前提下，补全个人中心页面，使其成为用户登录后的核心工作台。

------

# 一、设计原则

## 1. 风格统一原则

个人中心必须与当前主站保持一致，优先参考以下页面与组件：

- `HomeView.vue`
- `SpotListView.vue`
- `NavBar.vue`

不要参考：

- `LoginView.vue`
- `RegisterView.vue`

原因：
 登录注册页是偏 SaaS 的表单登录视觉，而个人中心属于主站内容体系的一部分，应该保持“文旅展示型产品”的视觉语言。

------

## 2. 页面头部原则

全局已有 `NavBar`，因此个人中心**不需要再做一个 Hero 级大页头**。
 但由于个人中心承载多个模块和副标题，**必须保留一个轻量内容头部**。

### 正确做法

在主内容区域顶部写一个轻量页面头：

- 标题：`个人中心`
- 副标题：`管理你的个人资料、活动报名与互动记录`

### 不允许做法

- 不做大背景图
- 不做整屏大页头
- 不做与首页 Hero 重复的视觉模块
- 不新增与导航重复的头部结构

------

## 3. 统一性原则

个人中心在以下方面必须与现有项目统一：

- 容器宽度
- 左右边距
- 品牌颜色
- 卡片圆角
- 按钮风格
- 弹窗风格
- 留白节奏
- 响应式逻辑

------

# 二、页面整体布局规范

## 1. 页面主容器

个人中心页面主容器统一采用主站内容宽度规范：

- `max-width: 1280px`
- `margin: 0 auto`
- `padding-left / right: 24px`
- 大屏可用 `40px`

推荐实现：

```
max-w-[1280px] mx-auto px-6 lg:px-10
```

### 禁止

- 不再使用当前 `ProfileView` 的 `max-w-2xl`
- 不做表单页式窄中轴布局

------

## 2. 页面纵向结构

页面结构从上到下：

1. 全局导航栏 `NavBar`
2. 个人中心内容区
   - 页面轻量头部
   - 双栏主体布局
3. 页面底部留白

------

## 3. 主体双栏布局

桌面端采用双栏：

- 左侧：用户信息卡
- 右侧：业务工作区

推荐尺寸：

- 左栏宽度：`300px ~ 320px`
- 右栏：`1fr`
- 栏间距：`24px`

推荐实现：

```
grid grid-cols-1 lg:grid-cols-[320px_minmax(0,1fr)] gap-6
```

------

## 4. 响应式规则

### 桌面端

- 双栏布局
- 收藏模块双列卡片

### 平板端

- 双栏可保留，但间距收缩
- 或提前切成纵向布局

### 移动端

- 单栏堆叠
- 顺序：
  1. 用户信息卡
  2. Tab
  3. 内容区

------

# 三、页面头部规范

## 1. 页面头部是否保留

保留，但必须轻量化。

## 2. 页面头部内容

- 标题：`个人中心`
- 副标题：`管理你的个人资料、活动报名与互动记录`

## 3. 视觉规格

- 标题字号：`28px`
- 字重：`700`
- 副标题字号：`14px ~ 16px`
- 副标题颜色：使用现有次级文本色
- 头部底部间距：`24px`

## 4. 风格约束

这个头部更像“内容标题条”，不是视觉大横幅。

### 不允许

- 大面积背景图
- 大块品牌色铺底
- 复杂插画
- 和首页重复的沉浸式头图

------

# 四、颜色系统统一规范

必须只使用现有项目里的设计 token，不新增新主色体系。

## 已有全局颜色

根据当前样式定义，优先使用：

- `--brand: #222a2c`
- `--light: #f5f4f2`
- `--muted: #648992`
- `--dark: #092c38`
- `--border: #dfdfdf`
- `--accent: #105670`

## 使用建议

### 页面背景

- 白色或极浅色背景

### 主正文

- `brand`

### 次级信息

- `muted`

### 左侧身份卡

- 使用 `brand / dark / accent` 体系渐变或纯深色

### 内容卡片

- 白底或浅色底
- 边框用 `border`

### 状态色

可新增语义色，但只能作为状态色，不可成为新主视觉：

- 成功：绿色
- 警告：橙黄
- 失败：红色

------

# 五、卡片与留白统一规范

## 1. 主卡片

用于：

- 左侧用户信息卡
- 右侧主内容容器

规格：

- 圆角：`24px`
- 内边距：`24px`
- 阴影：轻微
- 背景：白 / 浅色 / 深色品牌背景

## 2. 子卡片

用于：

- 我的资料字段块
- 报名记录项
- 收藏卡片
- 评论卡片

规格：

- 圆角：`16px ~ 20px`
- 内边距：`16px ~ 20px`
- 卡片间距：`16px`

## 3. 留白节奏

- 大区块之间：`24px ~ 32px`
- 标题与正文之间：`12px ~ 16px`
- 同类卡片之间：`16px`

### 禁止

- 密集堆叠
- 小圆角后台卡片风
- 过窄、过满布局

------

# 六、左侧用户信息卡规范

这是页面最强识别区，必须有品牌感。

## 1. 容器

- 圆角：`24px`
- 内边距：`24px`
- 背景：品牌深色渐变或深色纯色
- 文字：浅色高对比

## 2. 内容结构

按顺序：

1. 头像
2. 昵称
3. 用户名
4. 手机号
5. 邮箱
6. 状态标签
7. 操作按钮区

## 3. 头像

- 72px 圆形
- 有边框或浅色外圈
- 无头像时显示默认头像或昵称首字

## 4. 文案层级

- 昵称：主标题
- 用户名/手机号/邮箱：次级信息
- 状态标签：小胶囊

## 5. 操作按钮

- `编辑资料`：一期可禁用
- `退出登录`：次级危险操作

------

# 七、右侧工作区规范

右侧主内容区是一个统一大卡片容器。

## 1. 容器样式

- 白底
- 圆角 `24px`
- 内边距 `24px`

## 2. 结构

- 顶部：Tab 导航
- 下方：Tab 内容

------

# 八、Tab 规范

## Tab 项

- 我的资料
- 我的报名
- 我的收藏
- 我的评论

## 样式

- 胶囊式切换
- active 使用品牌色体系
- inactive 为浅灰底

## 交互

- 支持 URL query：
  - `?tab=info`
  - `?tab=applications`
  - `?tab=collections`
  - `?tab=comments`

## 移动端

- 横向滚动
- 不换行

------

# 九、四个 Tab 的统一风格要求

------

## 1. 我的资料

### 风格要求

- 更稳重、规整
- 像“账户信息区”，但不能太后台

### 布局

- 2 列字段卡片
- 每个字段单独成块
- 空值统一显示 `未填写`

------

## 2. 我的报名

### 风格要求

- 业务感最强
- 信息完整
- 状态表达清楚

### 样式

- 大卡片纵向列表
- 活动封面 + 信息 + 状态区

### 状态 badge

- 待审批：黄
- 已通过：绿
- 已拒绝：红

------

## 3. 我的收藏

### 风格要求

- 更轻快一些
- 更贴近内容浏览体验

### 布局

- 网格卡片
- 类型标签清晰
- 保留“取消收藏”按钮

------

## 4. 我的评论

### 风格要求

- 更强调互动记录
- 评论正文有明显内容区

### 布局

- 单列卡片
- 顶部活动信息
- 中部评论内容
- 底部互动信息

------

# 十、弹窗统一规范

## 1. 是否复用现有弹窗

复用 `ConfirmModal`，不需要新造。

## 2. 允许微调

为了更贴合主站，允许以下微调：

- 圆角：从 `12px` 调整到 `16px`
- 主按钮沿用品牌色
- 阴影更柔和
- 文案更完整

## 3. 退出登录弹窗文案

- 标题：`确认退出登录？`
- 内容：`退出后你仍可浏览公开内容，但无法继续进行报名、收藏与评论操作。`

## 4. 原则

弹窗仍保持简洁，不做视觉夸张改造。

------

# 十一、按钮统一规范

## 主按钮

- 使用品牌深色或 accent
- 圆角偏大
- hover 有轻微变化

## 次按钮

- 线框或浅底
- 颜色与品牌保持一致

## 危险按钮

- 仅退出登录、取消收藏等使用
- 不使用过于刺眼的纯红大块

------

# 十二、状态页统一规范

## Loading

- 必须用骨架屏
- 左侧资料卡和右侧内容都要有骨架

## Empty

必须包含：

- 图标/插画
- 主文案
- 副文案
- CTA

## Error

- 提示文案
- 重新加载按钮

## Partial

- 左栏成功、右栏失败时，只提示右栏，不全页报错

------

# 十三、与现有项目一致性结论

Codex 在实现时必须遵循这三个判断：

## 1. 风格

个人中心必须归入主站风格体系，不能做成独立后台页或 SaaS 页。

## 2. 页面头部

要保留一个轻量内容页头，用来承接副标题；
 但不能做重复 Hero，也不能与全局导航冲突。

## 3. 统一项

以下必须统一：

- 最大宽度：1280px
- 品牌色：只用现有 token
- 卡片圆角：24 / 16~20
- 弹窗：复用 ConfirmModal 微调
- 留白节奏：与主站一致
