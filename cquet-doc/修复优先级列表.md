# 二、修复优先级表

这个版本是给 Codex 直接排期的。

## 第 1 阶段：工程稳定 + 核心链路修复（必须先做）

**目标**：让项目能稳定跑、能登录、能报名、能公开浏览

| 编号 | 任务                                                        | 优先级 | 依赖 | 预计输出                   |
| ---- | ----------------------------------------------------------- | ------ | ---- | -------------------------- |
| T1   | 清理依赖环境，删除仓库内 `node_modules`，保证干净安装可运行 | P0     | 无   | 可稳定 `install/dev/build` |
| T2   | 统一 API 响应解析模型                                       | P0     | T1   | 通用响应类型与适配逻辑     |
| T3   | 修复登录 token 读取与用户信息初始化                         | P0     | T2   | 登录闭环打通               |
| T4   | 修复活动报名成功/失败判断                                   | P0     | T2   | 报名链路可用               |
| T5   | 统一 API baseURL 策略                                       | P1     | T1   | 环境配置一致               |
| T6   | 调整新闻/公告 401 处理策略，避免公开页直接跳登录            | P1     | T2   | 公开页更稳                 |

**阶段验收**

- 干净环境下能启动
- 登录成功后能进入 `/profile`
- 活动报名至少在前端逻辑层正确提示
- 新闻公告不因拦截器误跳登录
