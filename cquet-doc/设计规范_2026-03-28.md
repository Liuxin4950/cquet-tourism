## Problem Statement

毕业设计项目，需要为重庆文旅网站构建用户端界面。要求：
1. **技术栈**：Vue3 + Vite（而非昨天方案中的 React）
2. **视觉风格**：仿照 https://www.rangedevelopmentsgroup.com/（高端地产官网风格）
3. **内容**：重庆文旅（景区、场馆、活动），用真实重庆元素替代参考网站的地产内容
4. **交付**：3周，4月中旬前答辩

---

## What Makes This Cool

**高端质感 + 山城内容**——用做 luxury hotel 网站的手法做旅游平台。纯白背景（`#fff`）+ 深青灰主色（`#222a2c`）+ Manrope 字体 + 大幅实拍摄影（洪崖洞夜景、大足石刻、长江三峡）+ Swiper 轮播卡片 + GSAP 简单动画。评委看到的第一反应会是："这不是一个学生项目"。

## Premises

1. **Vue3 + Vite 技术栈** — 与现有管理端同语言，可共享样式变量；比 React 上手更快。**agree/disagree？**
2. **Range Developments 视觉风格** — 深蓝/青色色调 + Montserrat 字体 + 大幅摄影，适合做旅游平台的"高端感"。**agree/disagree？**
3. **内容替换而非全部仿照** — 保留参考网站的风格框架，内容换成重庆文旅（洪崖洞、火锅、大足石刻等），不做地产销售元素。**agree/disagree？**
4. **3周交付范围** — 首页 + 详情页模板 + 登录注册 + 用户中心；Swiper 轮播和 GSAP 动画保留为核心交互，mega menu 和地图联动暂不做。**agree/disagree？**

## Approaches Considered

### Approach A: 最小可行（最快交付）
**技术路径**：Vue3 + Vite + Tailwind CSS，纯 CSS 动画，无 GSAP/Swiper。

- 页面：首页（静态）+ 详情页（模板）+ 登录
- 工作量：约1.5周
- **Pros**：最快出活，风险最低
- **Cons**：动画质感差，参考网站风格还原度低，答辩可能不够惊艳

---

### Approach B: 保留核心（推荐）
**技术路径**：Vue3 + Vite + Tailwind CSS + GSAP（简单滚动动画）+ Swiper（轮播卡片）。

- **首页**：Hero（全屏背景图 + 标题动画）+ 精选推荐（Swiper 横向滚动）+ 关于重庆（双栏布局）+ 三栏入口 + Footer
- **详情页**：大图 Banner + 信息块 + 报名/收藏按钮
- **登录/注册**：JWT token 处理
- **用户中心**：收藏列表 + 报名记录 + 留言记录
- 工作量：约2.5-3周

**Pros**：
- 风格还原度高（色调/字体/布局/动画核心都在）
- Swiper 轮播已经是参考网站的核心元素
- Vue3 你更熟悉，3周内可交付
- 比 React deck.gl 方案风险低

**Cons**：
- GSAP 有学习曲线
- 需要收集重庆文旅真实图片素材

### GSAP "简单滚动动画"具体约定（必须实现）
1. **Hero 标题进入**：页面加载后，标题从 `opacity:0, y:30` → `opacity:1, y:0`，duration 0.8s，ease: power2.out
2. **精选推荐卡片**：滚动进入视口时，`opacity:0, y:40` → `opacity:1, y:0`，duration 0.6s，stagger 0.1s（使用 GSAP ScrollTrigger）
3. **关于重庆数字**：数字从 0 递增动画到目标值（countUp 效果，duration 1.5s）
4. **三栏卡片**：hover 时 `scale:1.02, y:-4`，transition 0.25s ease

> **Week 1 验证 Spike**：第一周第一天必须验证 Tailwind CSS transitions + GSAP ScrollTrigger + Swiper 三者共存无冲突。若有冲突，优先保留 Swiper（核心交互），GSAP 降级为纯 CSS 动画。

---

### Approach C: 完整仿照
**技术路径**：Vue3 + Vite + Tailwind CSS + GSAP（全套 ScrollTrigger）+ Swiper + SVG 地图 + Mega Menu。

- 完整还原参考网站的 mega menu、地图联动、award marquee 等所有元素
- 工作量：4-6周（超出 deadline）
- **Pros**：最完整
- **Cons**：工作量超出3周，答辩可能来不及

---

## Recommended Approach

**Approach B（保留核心）** — 在3周内交付有参考网站质感的完整旅游平台，同时保留 Swiper 轮播和基础 GSAP 动画作为核心记忆点。

## API Contract（基于真实后端）

### 已知后端已有接口（可直接对接）

#### 认证（若依框架自带）
| 接口 | 方法 | 说明 |
|------|------|------|
| `/login` | POST | 登录，body: `{ username, password }`，返回若依标准 `Result` 含 token |
| `/getInfo` | GET | 获取当前用户信息（含权限） |

> **注册**：后端**无**注册接口。若依默认只有管理员创建用户。**必须新增** `/register` 接口（预计 1-2 小时），或使用管理员账号创建测试用户用于答辩演示。

#### 景区（A级景区）
| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `GET /tourism/scenic-spot/list` | GET | `tourism:scenicSpot:list` | 列表，分页+名称搜索 |
| `GET /tourism/scenic-spot/{id}` | GET | `tourism:scenicSpot:query` | 详情（含 `images[]`） |
| `GET /tourism/scenic-spot/{id}/venues` | GET | — | 景区关联的场馆列表 |

> ⚠️ **权限问题**：以上接口均有 `@PreAuthorize` 权限限制。用户端访问时需先调用 `/login` 获取 token，后续请求在 header 携带 `Authorization: Bearer {token}`。若用户端面向公众（无需登录），需在若依安全配置中放行这些接口，或新增"用户端专用接口"（免权限版本）。

#### 场馆
| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `GET /tourism/venue/list` | GET | `tourism:venue:list` | 列表，参数：`name`、`address`、`city` |
| `GET /tourism/venue/{id}` | GET | `tourism:venue:query` | 详情（含 `images[]`、`facilities[]`） |
| `GET /tourism/venue/{id}/activities` | GET | `tourism:venueActivity:list` | 场馆下的活动列表 |

#### 特色活动
| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `GET /tourism/activity/list` | GET | `tourism:activity:list` | 列表，参数：`name`、`venueId`、`auditStatus` |
| `GET /tourism/activity/{id}` | GET | `tourism:activity:query` | 详情（含 `auditStatus`） |

#### 活动申报（用户报名）
| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `GET /tourism/activity-approval/list` | GET | `tourism:activityApproval:list` | 当前用户的报名列表（含审批状态） |
| `POST /tourism/activity-approval/{id}/pass` | POST | `tourism:activityApproval:pass` | 审核通过（管理员） |
| `POST /tourism/activity-approval/{id}/reject` | POST | `tourism:activityApproval:reject` | 审核拒绝（管理员） |

### 需要新增的后端接口

| 接口 | 方法 | 说明 | 预计工时 |
|------|------|------|---------|
| `POST /register` | POST | 用户注册（body: `{ username, password, nickName }`） | 1-2h |
| `GET /tourism/collection/list` | GET | 当前用户的收藏列表 | 2-3h |
| `POST /tourism/collection` | POST | 添加收藏（body: `{ targetType, targetId }`） | 1-2h |
| `DELETE /tourism/collection/{id}` | DELETE | 取消收藏 | 1h |
| `GET /tourism/comment/list` | GET | 活动留言列表（参数：`activityId`） | 2h |
| `POST /tourism/comment` | POST | 发表评论（body: `{ activityId, content, parentId? }`） | 2h |

> ⚠️ **优先级建议**：收藏和评论属于"锦上添花"功能，若时间紧张可延后至答辩后。用户中心可先做报名记录展示（已有 `activity-approval/list` 接口）。

### JWT Token 处理
- 若依将 token 放在 **cookie** 或 **header** `Authorization: Bearer {token}`
- 前端 Axios 拦截器统一注入：`request.interceptors.request.use(config => { config.headers['Authorization'] = 'Bearer ' + getToken(); return config; })`
- Vite dev：proxy 配置 `/tourism/**` → `localhost:8080`
- 生产：后端配置 CORS 允许前端域名；或前后端同域部署

### 图片处理
- 图片上传：`POST /common/upload`（已有，若依自带）
- 景区图片：`GET /tourism/scenic-spot/{id}/images`
- 场馆图片：`GET /tourism/venue/{id}/images`
- 前端轮播/详情页直接调用这些接口获取图片 URL 列表

"294个A级景区 · 38个场馆 · 126个特色活动"为静态展示数字，写死在首页代码中。
