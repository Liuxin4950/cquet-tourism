## 1. 概述

### 1.1 项目简介

重庆文旅网站管理系统（cquet-demo）包含以下模块：

- **后端管理**：基于若依框架开发的管理后台（Vue 2 + Element UI）
- **用户前端**：独立的现代化旅游网站前端（Vue 3 + Vite）
- **AI集成**：计划使用 SpringAI + MCP 实现智能景区推荐

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Spring Boot 3.3.2 + MyBatis-Plus |
| 安全框架 | Spring Security + JWT |
| API 文档 | Swagger/OpenAPI（`@Tag/@Operation` 注解） |
| 分页 | PageHelper + MyBatis-Plus |
| 构建工具 | Maven |

### 1.3 基础路径约定

| 环境 | 基础路径 |
|------|----------|
| 开发环境 | `http://localhost:8080` |
| 后端 API 前缀 | 无前缀（直接 `/controller-path`） |
| 用户端前端 | `http://localhost:5173` |

## 2. 接口规范

### 2.1 认证模块

#### 2.1.1 用户登录

```
POST /login
```

**请求体**：

```json
{
  "username": "string",
  "password": "string",
  "code": "string (验证码，可选)",
  "uuid": "string (验证码ID，可选)"
}
```

**响应（成功）**：

```json
{
  "code": 200,
  "msg": "success",
  "state": true,
  "data": null,
  "time": 1743400000000,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**说明**：
- 返回的 `token` 需在后续请求的 `Authorization` header 中携带：`Bearer {token}`
- 验证码相关参数在系统开启验证码校验时必传

---

#### 2.1.2 用户注册

```
POST /register
```

**请求体**：继承自 `LoginBody`，同登录

```json
{
  "username": "string",
  "password": "string",
  "code": "string (可选)",
  "uuid": "string (可选)"
}
```

**响应**：

```json
{
  "code": 200,
  "msg": "注册成功",
  "state": true
}
```

或

```json
{
  "code": 500,
  "msg": "当前系统没有开启注册功能！",
  "state": false
}
```

**说明**：
- 需要系统配置 `sys.account.registerUser = true` 才可注册
- 注册后自动分配默认角色

---


### 2.2 A级景区模块

> **注意**：以下接口均已标记为 `@Anonymous`，无需登录即可访问（用户端公开访问）。

#### 2.2.1 景区列表

```
GET /tourism/scenic-spot/list
```

**权限**：公开（无需认证）

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 否 | 景区名称（模糊查询） |
| city | string | 否 | 所在城市 |
| level | string | 否 | 景区等级（如 "4A", "5A"） |
| pageNum | int | 否 | 页码（默认 1） |
| pageSize | int | 否 | 每页条数（默认 10） |

**响应**：

```json
{
  "total": 100,
  "rows": [
    {
      "id": 1,
      "name": "洪崖洞民俗风貌区",
      "level": "4A",
      "city": "重庆市",
      "address": "重庆市渝中区解放碑商圈",
      "ticketPrice": 0,
      "openTime": "08:00-22:00",
      "description": "国家4A级旅游景区...",
      "status": "1"
    }
  ],
  "code": 200,
  "msg": "查询成功"
}
```

---

#### 2.2.2 景区详情

```
GET /tourism/scenic-spot/{id}
```

**权限**：公开（无需认证）

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | long | 景区ID |

**响应**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "name": "洪崖洞民俗风貌区",
    "level": "4A",
    "city": "重庆市",

    "ticketPrice": 0,
    "openTime": "08:00-22:00",
    "description": "国家4A级旅游景区...",
    "status": "1",
    "images": [
      {"id": 1, "url": "https://.../hongyadong1.jpg"},
      {"id": 2, "url": "https://.../hongyadong2.jpg"}
    ],
    "venues": [
      {"id": 1, "name": "洪崖洞会议室"}
    ]
  }
}
```

**异常响应**：

```json
{
  "code": 301,
  "msg": "景区不存在",
  "state": false
}
```

#### 2.2.3 新增景区

```
POST /tourism/scenic-spot
```

**权限**：`tourism:scenicSpot:add`

**请求体**：

```json
{
  "name": "string",
  "level": "string",
  "city": "string",
  "address": "string",
  "ticketPrice": "number",
  "openTime": "string",
  "description": "string",
  "status": "string"
}
```

**说明**：
- `id` 不需要指定（自动生成）
- 若指定 `id` 会返回警告

**响应**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 10,
    "name": "新景区名称"
  }
}
```

---

#### 2.2.4 修改景区

```
PUT /tourism/scenic-spot
```

**权限**：`tourism:scenicSpot:edit`

**请求体**：同新增，必须指定 `id`

```json
{
  "id": 10,
  "name": "修改后的名称",
  "level": "5A"
}
```

**响应**：

```json
{
  "code": 200,
  "msg": "success"
}
```

## 3. 通用约定

### 3.1 认证方式

- **JWT Token**：登录成功后获取，存于响应 `token` 字段
- **携带方式**：请求 Header 添加 `Authorization: Bearer {token}`
- **有效期**：由若依框架配置决定（默认 30 分钟，可刷新）
- **公开访问**：部分旅游业务接口（景区/场馆/活动列表和详情）已标记 `@Anonymous`，无需 token 即可访问。管理操作（增删改）仍需认证。
