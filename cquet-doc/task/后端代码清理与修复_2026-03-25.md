## 2. 任务清单

### 2.1 补全 ActivityApplication 后端（缺失）

**现状：** 数据库有 `tourism_activity_application` 表，前端有 `activityApplication` 页面，但后端 Java 代码完全缺失。

**需要创建：**
- `TourismActivityApplication.java` (Entity)
- `TourismActivityApplicationMapper.java` (Mapper)
- `TourismActivityApplicationService.java` + `TourismActivityApplicationServiceImpl.java` (Service)
- `TourismActivityApplicationController.java` (REST API)

**API 端点：**
- `GET /list` — 报名列表（分页 + 过滤）
- `GET /{id}` — 报名详情
- `POST /` — 用户提交报名
- `PUT /{id}/status` — 审核通过/拒绝
- `DELETE /{id}` — 取消报名

**操作：** 从数据库删除该表（SQL 命令）。Java 代码中没有使用（已确认）。

**操作：** 删除该文件。

## 3. 技术方案

### 3.1 ActivityApplication 后端

遵循现有项目结构：
- 继承 MyBatis-Plus `BaseMapper`
- Service 继承 `IService<T>`
- Controller 继承 `BaseController`
- 使用 `@PreAuthorize` 权限控制
- 使用 `@Log` 记录操作

### 3.2 数据库修改

执行 SQL：
```sql

```
